Project Overview
The Vehicle CAN Bus MF4 Decoding & Analytics Pipeline is designed to process large binary MF4 files generated during pre-launch vehicle testing.During testing, the vehicle’s ECU communicates with sensors via the CAN bus (binary messaging system). High-frequency CAN signals such as RPM, speed, brake status, and engine temperature are captured and stored in MF4 binary format (4–5 GB per file). These files are not human-readable and require DBC files for decoding.
This project automates the complete lifecycle:
  • Validating MF4 file integrity
  • Verifying DBC version compatibility
  • Orchestrating serverless decoding workloads
  • Streaming-based decoding using containerized processing
  • Enforcing strict schema validation
  • Managing failures using DLQ and retry workflows
  • Preparing analytics-ready Parquet datasets for querying
